# Spanish Pilot Selection Criteria v1

## Purpose
Select the highest-confidence pilot book candidate for the first Spanish edition lane while preserving canonical English authority in source Book repos.

## Scoring Method
- Score each criterion from 1 (low readiness) to 5 (high readiness).
- Weight applies to operational impact.
- Candidates with any critical red flag in rights, claims, or reviewer availability are held regardless of total score.

| Criterion | Weight | What Good Looks Like |
|---|---|---|
| English manuscript maturity | 5 | Most chapters are stable and approaching or at `EN_TRANSLATION_READY`. |
| Glossary readiness | 5 | Core terminology exists with low unresolved ambiguity for EN->ES transfer. |
| Claim/source stability | 5 | Citations and claims are stable, with minimal unresolved disputes. |
| Technical complexity | 3 | Complexity is manageable for pilot throughput and quality controls. |
| Audience value for Spanish edition | 5 | Clear Spanish reader demand and practical user impact. |
| Expected education/workforce usefulness | 4 | Content has strong direct value for training, education, or workforce outcomes. |
| Formatting complexity | 3 | Tables/figures/layout burden is feasible for pilot cadence. |
| Rights/metadata readiness | 5 | Rights are clear and metadata + ISBN prep can proceed without blockers. |
| Likelihood of becoming reusable process proof | 4 | Execution is likely to produce repeatable SOPs and templates. |
| Risk level | 5 | Low-to-moderate operational/legal/editorial risk profile. |
| Reviewer availability | 5 | Qualified terminology, native-language, and technical reviewers are available. |

## Selection Rules
- Minimum recommended total score: 44 of 60 weighted-equivalent readiness points.
- Mandatory pass criteria:
- Rights/metadata readiness must be >= 4.
- Claim/source stability must be >= 4.
- Reviewer availability must be >= 4.
- English manuscript maturity must indicate near-term `EN_TRANSLATION_READY` intake feasibility.

## Decision Outcomes
- `RECOMMEND`: Candidate can enter intake packet preparation.
- `HOLD`: Candidate needs remediation before pilot admission.
- `REJECT_FOR_PILOT_WINDOW`: Candidate is not suitable for this tranche.
