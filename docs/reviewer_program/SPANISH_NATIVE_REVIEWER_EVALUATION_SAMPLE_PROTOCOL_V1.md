# Spanish Native Reviewer Evaluation Sample Protocol V1

## Purpose

Define a lightweight sample protocol for qualifying a native-language reviewer before assigning the Book 1 Spanish pilot review. The protocol evaluates reviewer fit without treating the sample as the actual native review.

## Protocol Status

HOLD_FOR_OPERATOR_REVIEWER_SELECTION

## Sample Selection

The operator may provide a short sample from the Spanish draft or a comparable non-confidential Spanish review sample. If project text is used, preserve confidentiality/IP controls and do not duplicate full manuscript or full draft material into this protocol.

## Sample Review Tasks

The candidate reviewer should demonstrate ability to:

- identify unnatural Spanish.
- flag overly literal phrasing.
- preserve meaning without adding unsupported claims.
- identify omissions or additions.
- comment on tone/register for education and workforce readers.
- recognize terminology that should remain consistent.
- flag regional wording if neutral Spanish is expected.
- state when a concern belongs to technical review rather than native-language review.

## Acceptance Criteria

Accept the candidate only if the sample shows reliable language judgment, source-alignment discipline, documentation clarity, and respect for confidentiality/IP and AI-use boundaries.

## Rejection Or Hold Criteria

Hold or reject the candidate if the sample shows repeated meaning drift, unsupported additions, inability to flag literal phrasing, unwillingness to document findings, or refusal to follow confidentiality/IP or AI-use boundaries.

## Boundary

Passing this sample protocol would only support reviewer assignment. It would not complete native-language review, human review, technical review, final review, publication readiness, release approval, or upload readiness. Publication remains blocked until documented native review occurs and all downstream gates pass.
